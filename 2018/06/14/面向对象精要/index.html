引用类型

鉴别引用类型

JavaScript 的值可以在同一个网页的不同框架间传来传去。这会成为鉴别引用类型时候的问题。因为每个页面拥有它自己的全局上下文，Object 和 Array 以及其他内建类型的版本。结果，当我们把一个数组从一个框架传到另一个框架时，instanceof 就无法识别它，因为那个数组来自不同框架的 Array 实例。因此推荐采用 Array.isArray() 方法鉴别数组

原始封装类型

三种：String 、Number 和 Boolean

举例说明内部机制

🌰 实际上 JS 引擎创建了一个字符串对象让方法可以调用执行。字符串对象 的存在仅用于方法执行的那一句，之后便被销毁。

🍐 实际上是在一个立刻就被销毁的临时对象上添加了新属性。访问新属性的 时候另一个不同的临时对象被建立，而在这个对象上，不存在新属性。

🍎 虽然原始封装对象会自动创建，但是进行 instanceof 检查时，返回均 为 false。因为临时对象仅仅在值被读取的时候创建。该操作符并没有 读取，也就无临时对象创建。

原始封装类型的副作用

函数

使得函数不同于其他对象的特点是函数存在一个 [[Call]] 的内部属性。该属性函数独有，表明该对象（函数也是对象）可以被执行。正因为该属性为函数独有，所以 typeof 操作符对任何具有 [[Call]] 属性的对象返回 "function"

内部属性无法通过代码访问，而是定义了代码执行时的行为。
JavaScript 对象有多种内部属性，这些内部属性都用双重中括号标注。

声明还是表达式

函数声明会将函数进行提升，提升至上下文（要么是该函数被声明时所在的函数的范围；要么是全局范围）顶部。
函数表达式不会进行提升。

参数

  1. arguments 对象不是一个数组的实例， Array.isArray(arguments) 永远返回 false

  2. 命名参数的影响，函数期望的参数个数保存在函数的 length 属性中。实际传入的参数的数量不影响函数的期望参数个数。

重载

  1. 函数签名：函数名、参数个数及参数类型
  2. 可以通过 typeof 、instanceof 以及 arguments 实现函数重载
  3. 声明两个同名函数，第一个会丢失，只使用最后那个：

理解对象

定义属性

  1. 当一个属性第一次被添加给对象时，JavaScript 会在对线上调用一个 [[Put]] 内部方法，该方法在对象上创建一个新节点来保存属性。这个操作不仅指定了初始值，而且定义了属性的一些特征。

    调用 [[Put]] 的结果是在对象上创建了一个自有属性。这表明仅仅该指定的对象实例拥有这个属性。该属性被直接保存在实例内部，对该属性的所有操作必须通过该对象进行

  2. 当一个已有属性被赋予新值时，调用名为 [[Set]] 的内部方法，该方法将属性的当前值替换为新值。

属性探测

  1. in 操作符：检查自有属性和原型属性

  2. hasOwnProperty() 仅仅检查自有属性

删除属性

delete 操作符针对单个对象属性调用 [[Delete]] 的内部方法。删除成功时候，返回 true

属性枚举

  1. 所有我们添加的属性默认都是可枚举的，即可以用 for-in 循环遍历。可枚举属性的内部特征 [[Enumerable]] 都被设置为 true

  2. Object.keys() 返回属性数组，属性需是可枚举的

  3. for-in 会遍历原型属性和自有属性
  4. Object.keys() 只会遍历自有属性
  5. 并非所有属性可枚举！对象的大多数原生方法的 [[Enumerable]] 特征都被置为 false
  6. 可以用 propertyIsEnumerable() 检查属性是否可枚举

属性类型

数据属性 和 访问器属性

  1. [[Put]] 方法默认创建数据属性

  2. 访问器属性定义了 getter 和 setter 函数，访问器属性仅需要二者中任意一个，也可以二者都有

    仅定义 getter，属性变为只读；
    仅定义 setter，属性变为只写。

属性特征

通用特征：

数据和访问器属性都具有的两个特征：[[Enumerable]] 和 [[Configurable]]

  1. 可以用 delete 删除一个可配置的属性，或随时改变它，也可以把一个数据属性改变为访问器属性，或者相反。
  2. 声明的所有属性默认都是可枚举的、可配置的。
  3. 改变属性特征 Object.defineProperty()

数据属性特征：

数据属性独有特征 [[Value]] 该特征包含属性的值；[[Writable]] 指示该属性是否可以写入，所有的属性默认都是可写的。

  1. 当使用 Object.defineProperty() 定义新的属性时候，一定要为所有的特征指定一个值，否则布尔类型的值就会被默认设置为 false
  2. 当使用 Object.defineProperty() 改变一个已有属性时候，只有指定的特征才会被改变

访问器属性特征：

访问器属性独有特征 [[Get]] 和 [[Set]] 。两者定义一个，即可创建一个访问器属性。

  1. 如果创建一个同时具有访问器特征和数据特征的属性，会报错
  2. 试图读取一个没有 getter 的访问器属性总是会返回 undefined
  3. 严格模式下写入没有 setter 的访问器属性会抛出错误；非严格模式下会失败

定义多重属性：

Object.defineProperties()

获取属性特征：

Object.getOwnPropertyDescriptor() 只可用于自有属性。如果属性存在，返回一个描述对象。

禁止修改对象

对象也有指导其行为的内部特征，[[Extensible]] 是一个布尔值，指明该对象本身是否可以被修改。如果为 false 即可禁止新属性的添加。

应确保对禁止扩展对象、被封印对象、被冻结对象使用严格模式，得到及时报错信息

禁止扩展：

  1. Object.preventExtensions() 创建一个不可扩展的对象。不能再给传入其中的对象添加新的属性。
  2. Object.isExtensible() 检测 [[Extensible]] 值

对象封印：

一个被封印的对象不可扩展且所有属性不可配置。即，不能添加新属性，不能删除属性或改变属性类型（从数据属性到访问器属性，或反之）。
如果一个对象被封印，只能读写它的属性。

  1. Object.seal() 封印一个对象，调用时，[[Extensible]] 被置为 false ，其所有属性的 [[Configurable]] 特征被置为 false 。
  2. Object.isSealed() 判断一个对象是否被封印。

对象冻结：

一个被冻结的对象不能在其上添加或删除属性，不能改变属性类型，不能写入任何数据。简而言之，被冻结的对象是一个数据属性都为只读的被封印对象。
被冻结对象无法解冻。

  1. Object.freeze() 冻结对象
  2. Object.isFrozen() 判断对象是否被冻结

对象冻结极少被使用

构造函数和原型对象

构造函数

检查对象的类型：

  1. 每个对象创建时，都自动拥有一个构造函数属性，其中包含了一个指向其构造函数的引用。
  2. 通过对象字面量形式或 Object 构造函数创建出来的对象，其构造函数属性指向 Object
  3. 构造函数可以被覆盖，不一定准确，因此应该使用 instanceof
  4. 确保使用 new 调用构造函数

原型对象

  1. 几乎所有的函数都有一个名为 prototype 的属性，该属性是一个原型对象，用来创建新的对象实例。
  2. 鉴别原型属性

[[Prototype]] 属性

  1. 一个对象实例通过内部属性[[Prototype]] 跟踪其原型对象。该属性是一个指向该实例使用的原型对象的指针。new 一个对象时，构造函数的原型对象会被赋给该对象的 [[Prototype]] 属性。
  2. 对象的 __proto__ 属性可以直接读写 [[Prototype]] 属性
  3. isPrototypeOf() 检查某个对象是否是另一个对象的原型对象
  4. 自有属性会覆盖原型属性
  5. 无法删除一个对象的原型属性
  6. 原型对象中存储引用类型数据要注意

  7. 直接用对象字面量替换原型对象，会改变原型对象的 constructor 属性，需要再次指定构造函数属性

    对象实例和构造函数之间没有直接联系，但是对象实例和原型对象以及原型对象和构造函数之间有直接联系。如果打断对象实例和原型对象间的联系，那么也会打断对象实例与构造函数的联系。

  8. 当在对象被封印或被冻结时，对自有属性有限制，但是仍然可以通过在原型对象上添加属性来扩展对象实例。因为 [[Prototype]] 属性是对象实例的自有属性，属性本身被冻结，但是其指向的值（原型对象）并没有被冻结。

    不建议通过修改内建对象（如：Array 、String 等）的原型对象达到扩展功能的目的。

继承

Object.prototype 的方法

  1. valueOf()
    每当一个操作符被用于一个对象时，就会调用该方法。该方法默认返 回对象实例本身。但可以对其进行重写，例如 Date 对象的该方法返回一个时间戳。

  2. toString()
    一旦 valueOf() 方法返回的是一个引用而不是原始值的时候，就 会调用 toString() 方法。
    另外，当 JavaScript 期望一个字符串时，也会对原始值隐式调用 toString()

  3. 不建议在 Object.prototype 上添加属性或方法

对象继承

指定哪个对象是新对象的 [[Prototype]]

  1. Object.create()

  2. Object.prototype 的 [[Prototype]] 是 null

    创建的这种没有原型对象链的对象，完全是一个没有任何预定义属性的白板，这使得它成为一个完美的哈希容器，不会发生跟继承来的属性名字的冲突。

构造函数窃取

如果需要在子类的构造函数中调用父类的构造函数，那么需要利用 call() 和 apply() ，并将新创建的对象传入，即：用自己的对象窃用父类的构造函数。

这可以避免在构造函数里重新定义希望继承的属性。

伪类继承：修改 prototype 来继承方法，并用构造函数窃取来设置属性

调用父类方法

如果子类提供的新功能覆盖了父类的方法，那么如果要访问父类的方法，需要使用 call() 或 apply() 调用父类的原型对象的方法，并在这时传入子类的对象。

对象模式

私有成员和特权成员

模块模式：

一种用于创建拥有私有数据的单件对象的模式。利用立调函数表达式（IIFE）返回一个对象。

暴露模块模式：
将所有变量和方法都组织在 IIFE 的顶部，然后将它们设置到需要被返回的对象上。

构造函数的私有成员：

模块模式适合在定义单个对象的私有属性上十分有效。如果自定义类型也需要私有属性，那么可以在构造函数中使用类似模式来创建每个实例的私有数据。

如果需要所有实例可以共享的私有数据（就好像它被定义在原型对象里一样）

混入

一个对象在不改变原型链的情况下得到了另一个对象的属性被称为混入。第一个对象（接收者）通过直接复制第二个对象（提供者）的属性从而接收了这些属性。

这里是浅拷贝，如果属性内包含对象，那么提供者和接收者将指向同一个对象。

提供者的访问器属性在混入后，会变成接收者的数据属性。这是因为接收者的属性是被赋值语句，而不是由 Object.defineProperty() 创建。

作用域安全的构造函数

很多内建构造函数，例如：Array、RegExp 不需要 new 也可以工作，这是因为它们是作用域安全的构造函数。一个作用域安全的构造函数有没有 new 都可以工作，并返回同样类型的对象。
当使用 new 调用一个函数时候，this 指向的新创建的对象已经属于该构造函数所代表的自定义类型，即，可以在函数内使用 instanceof 来检查自己是否被 new 调用。
